# Final Project — Mac Team (MongoDB + PySpark)

This covers the **Mac team's scope** of the final project: ingest the dataset
into MongoDB Atlas, analyze it with PySpark, and export results for the
Windows team (SQL Server + Power BI).

## Architecture

```
Raw Dataset (CSV)
   |
MongoDB Atlas
   |
PySpark (EDA, aggregations, correlation, ML)
   |
processed_results (CSV + MongoDB collection)
   |
SQL Server / Power BI (Windows team)
```

## Dataset

`data/raw_dataset.csv` — a 10,000-row, stratified sample (by `default`) of
the cleaned LendingClub loan dataset, generated by `data/prepare_dataset.py`
from the Loan Default Predictor project's `loans_cleaned.parquet`. 25
columns: loan terms, borrower info (income, FICO, DTI, etc.), and the
`default` label (~21% positive rate).

## Setup

### 1. Python environment

PySpark 3.5.1 / pyarrow 16.1.0 don't yet have prebuilt wheels for Python 3.14,
so the project venv uses **Python 3.11** (installed via Homebrew):

```bash
/opt/homebrew/bin/python3.11 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

### 2. MongoDB Atlas (manual — do this in your browser)

1. Create a free [MongoDB Atlas](https://www.mongodb.com/cloud/atlas/register) account
2. Create an **M0 (free tier)** cluster
3. Create a database user (username + password)
4. Network Access → add your IP (or `0.0.0.0/0` for the demo)
5. Get the connection string: "Connect" → "Drivers" →
   `mongodb+srv://<user>:<password>@<cluster>.mongodb.net/?retryWrites=true&w=majority`

### 3. Configure `.env`

```bash
cp .env.example .env
```

Edit `.env` and paste in your real `MONGODB_URI`. Defaults:
- `MONGODB_DB=final_project`
- `MONGODB_COLLECTION=loans`
- `MONGODB_RESULTS_COLLECTION=processed_results`

## Running the Pipeline

```bash
# 1. (Already done) Prepare the dataset
.venv/bin/python data/prepare_dataset.py

# 2. Load data into MongoDB Atlas
.venv/bin/python ingestion/load_mongodb.py

# 3. Verify PySpark <-> MongoDB connection
.venv/bin/python spark/connect_mongodb.py

# 4. Run the full analysis (writes to outputs/)
.venv/bin/python spark/analysis.py
```

## Outputs

`spark/analysis.py` writes to `outputs/`:
- `summary_statistics.csv` — `describe()` on numeric columns
- `aggregation_by_grade.csv` — avg interest rate, avg loan amount, default rate by loan grade
- `aggregation_by_purpose.csv` — loan count, total amount, default rate by purpose
- `aggregation_by_home_ownership.csv` — loan count, avg income, default rate by home ownership
- `correlation_matrix.csv` — correlations between key numeric columns and `default`
- `model_metrics.txt` — logistic regression AUC/accuracy predicting `default` (bonus)
- `processed_results.csv` — the gold table (per-grade summary) for Power BI, also written
  back to MongoDB as the `processed_results` collection
